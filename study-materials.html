Domain 1: General Security Concepts (12%)

Key Topics:

  • Compare and contrast various types of security controls
  • Explain the importance of applicable regulations, standards, or frameworks
  • Explain the importance of policies to organizational security
  • Summarize risk management processes and concepts
  • Explain privacy and sensitive data concepts
  • Compare and contrast security implications of different architecture models

Study Notes:

Security Controls
Preventive Controls

Preventive controls are designed to stop security incidents before they occur. These include:

  • Firewalls: Network security devices that monitor and control incoming and outgoing network traffic
  • Encryption: Converting data into a code to prevent unauthorized access
  • Access Controls: Mechanisms that limit access to resources
  • Security Policies: Documented rules and procedures for security
  • Physical Security: Measures to protect physical assets
Detective Controls

Detective controls identify security incidents after they occur:

  • Intrusion Detection Systems (IDS): Monitor network traffic for suspicious activity
  • Log Monitoring: Review system logs for security events
  • Security Audits: Regular reviews of security controls
  • Video Surveillance: Monitor physical security
  • SIEM Systems: Collect and analyze security data
Corrective Controls

Corrective controls restore systems after security incidents:

  • Backup Systems: Restore data after loss
  • Incident Response Plans: Guide recovery efforts
  • Patch Management: Update systems to fix vulnerabilities
  • Disaster Recovery: Restore operations after major incidents
  • Business Continuity: Maintain critical operations
Deterrent Controls

Deterrent controls discourage potential attackers:

  • Warning Signs: Indicate security measures in place
  • Security Training: Educate users about security
  • Visible Security: Show security presence
  • Security Guards: Physical security presence
  • Security Cameras: Monitor and deter unauthorized access
Risk Management
Risk Assessment

Risk assessment involves identifying, analyzing, and evaluating risks:

  • Qualitative Assessment: Uses descriptive scales (e.g., High, Medium, Low)
  • Quantitative Assessment: Uses numerical values and calculations
  • Asset Valuation: Determine value of protected resources
  • Threat Modeling: Identify potential threats
  • Vulnerability Assessment: Find system weaknesses
Risk Treatment

Risk treatment options include:

  • Risk Acceptance: Acknowledge and monitor the risk
  • Risk Transfer: Shift risk to another party (e.g., insurance)
  • Risk Mitigation: Implement controls to reduce risk
  • Risk Avoidance: Eliminate the risk entirely
  • Risk Sharing: Partner with others to manage risk
Risk Monitoring

Continuous monitoring of risks includes:

  • Regular Reviews: Periodic risk assessments
  • Control Testing: Verify effectiveness of controls
  • Incident Tracking: Monitor security incidents
  • Compliance Checks: Ensure regulatory requirements
  • Performance Metrics: Measure control effectiveness
Regulations and Standards
ISO 27001

Information Security Management System (ISMS) standard:

  • Risk Assessment: Identify and evaluate risks
  • Security Controls: Implement appropriate controls
  • Continuous Improvement: Regular review and updates
  • Documentation: Maintain security policies and procedures
  • Certification: Independent verification of compliance
NIST Cybersecurity Framework

Framework for improving critical infrastructure cybersecurity:

  • Identify: Understand security risks
  • Protect: Implement safeguards
  • Detect: Identify security events
  • Respond: Take action on security incidents
  • Recover: Restore capabilities after incidents
GDPR

General Data Protection Regulation requirements:

  • Data Subject Rights: Individual control over personal data
  • Data Protection: Implement security measures
  • Breach Notification: Report data breaches
  • Privacy Impact: Assess data processing risks
  • Compliance: Meet regulatory requirements

Domain 2: Threats, Vulnerabilities, and Mitigations (22%)

Key Topics:

  • Compare and contrast common threat actors and motivations
  • Explain common threat vectors and attack surfaces
  • Explain various types of vulnerabilities
  • Given a scenario, analyze indicators of malicious activity
  • Explain the security concerns associated with various types of vulnerabilities
  • Summarize the techniques used in security assessments

Study Notes:

Threat Actors
Nation-State Actors

Government-sponsored attackers with significant resources:

  • Advanced Persistent Threats (APTs): Long-term targeted attacks
  • Cyber Espionage: Stealing sensitive information
  • Critical Infrastructure: Targeting essential services
  • Intellectual Property: Stealing trade secrets
  • Political Influence: Manipulating public opinion
Hacktivists

Attackers motivated by political or social causes:

  • Website Defacement: Altering website content
  • DDoS Attacks: Overwhelming systems with traffic
  • Information Leaks: Releasing sensitive data
  • Social Media: Manipulating online platforms
  • Data Breaches: Exposing confidential information
Organized Crime

Criminal groups focused on financial gain:

  • Ransomware: Encrypting data for ransom
  • Credit Card Fraud: Stealing payment information
  • Identity Theft: Stealing personal information
  • Banking Trojans: Stealing banking credentials
  • Cryptocurrency Theft: Stealing digital currency
Attack Vectors
Email Attacks

Common email-based attack methods:

  • Phishing: Deceptive emails to steal information
  • Spear Phishing: Targeted phishing attacks
  • Business Email Compromise: Impersonating executives
  • Whaling: Targeting high-profile individuals
  • Malicious Attachments: Infected email attachments
Malware

Types of malicious software:

  • Viruses: Self-replicating malware
  • Worms: Network-spreading malware
  • Trojans: Disguised malicious software
  • Ransomware: Data-encrypting malware
  • Spyware: Information-stealing malware
Social Engineering

Manipulating people to gain access:

  • Pretexting: Creating false scenarios
  • Baiting: Offering something enticing
  • Quid Pro Quo: Offering something in exchange
  • Tailgating: Following authorized personnel
  • Piggybacking: Gaining access with others
Vulnerability Management
Vulnerability Scanning

Identifying system weaknesses:

  • Network Scanning: Checking network devices
  • Web Application Scanning: Testing web apps
  • Database Scanning: Checking database security
  • Container Scanning: Testing container security
  • Cloud Scanning: Checking cloud infrastructure
Patch Management

Keeping systems up to date:

  • Vulnerability Assessment: Identifying needed patches
  • Patch Testing: Verifying patch compatibility
  • Deployment Scheduling: Planning patch rollout
  • Rollback Procedures: Reverting failed patches
  • Change Management: Documenting changes
Configuration Management

Maintaining secure configurations:

  • Secure Baselines: Standard secure configurations
  • Change Control: Managing configuration changes
  • Version Control: Tracking configuration versions
  • Documentation: Recording configurations
  • Compliance Monitoring: Ensuring security standards
